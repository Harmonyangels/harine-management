# Healthcare Blog Copywriting Skill
## Harine Management — harinemanagement.com

Read this entire file before writing any blog post for Harine Management.

## AUDIENCE
Primary: Medical practice owners, CMOs, practice administrators, and office managers at independent or PE-backed medical groups with 2–20 providers. Secondary: PE associates and healthcare investors evaluating or managing practice portfolio companies. These are busy, skeptical, operationally-minded people. They will stop reading the moment something sounds like marketing. Write like a smart colleague, not an agency.

## VOICE & TONE
- Direct. Say the thing. No warming up, no throat-clearing.
- Data-first. Every claim needs a number or a source (MGMA, HFMA, AAPC, CMS).
- No fluff. Cut any sentence that doesn't add information.
- Use correct clinical/billing terminology (A/R, wRVU, net collection rate, denial rate, payer mix).
- Never use: 'In today's fast-paced healthcare landscape', 'leverage', 'synergy', 'holistic approach', 'game-changer', 'revolutionary'
- Contractions are fine. First person is fine for CTA sections.

## POST STRUCTURE
1. Hook (1 paragraph) — open with a specific surprising benchmark stat with source. Format: '[Stat]. [One sentence context]. [One sentence transition to post topic].'
2. The Problem (2–3 paragraphs) — describe the specific operational or financial problem. Be concrete. Name the pain. No solutions yet.
3. The Benchmark (1–2 paragraphs + infographic) — state the industry benchmark, cite the source, explain what it means practically. Place [Infographic] component here.
4. What Good Looks Like (2–3 paragraphs) — describe a well-run practice on this metric. Use ranges not single numbers. Acknowledge specialty matters.
5. Mid-Post CTA — insert <ScoreCTA /> component exactly, do not write as prose.
6. How to Improve (3–5 points) — numbered list, bold title, 2 sentences each. No vague advice.
7. The Analytics Angle (1–2 paragraphs) — connect topic back to data visibility. Bridge to bottom CTA.
8. Bottom CTA — insert <DiscoveryCTA /> component exactly, do not write as prose.

## INTERNAL LINKING RULES
- Include 3–5 internal links per post minimum
- Link naturally within sentence context — never 'click here'
- Always link to /score at least once in body text
- Link to a relevant service page when topic is directly covered there
- Link to other blog posts when a concept is introduced that another post covers

## BENCHMARK SOURCES
Always cite one of these — do not make up statistics: MGMA, HFMA, AAPC, CMS, AMA.

## SEO RULES
- Target keyword must appear in: title, first paragraph, one H2, meta description
- Meta description: exactly 150–160 characters, includes target keyword, ends with a benefit statement
- H2s: 3–5 per post, written as questions or clear statements a searcher would use

## CATEGORIES (use exactly one)
Billing & RCM, Volume & Operations, Revenue Intelligence, PE & Investors, Athena Health

## EVERY POST MUST INCLUDE
- A benchmark stat with source in the first paragraph
- The <ScoreCTA /> component mid-post
- The <DiscoveryCTA /> component at the bottom
- At least one link to /score in body text
- At least one link to a relevant service page
- An SVG infographic visualizing the key stat or benchmark
- Frontmatter with all required fields complete
- Read time estimate in frontmatter

## NEVER DO
- Never invent benchmark numbers — only use real cited figures
- Never promise specific financial outcomes
- Never name competitors or specific EHR vendors negatively
- Never use patient data or identifiable practice information
- Never write more than 1,200 words
